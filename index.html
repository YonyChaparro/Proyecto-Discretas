Este proyecto busca aplicar los principios de las Matemáticas Discretas I, específicamente los temas de conteo, a un problema relevante en Ingeniería de Sistemas y Computación: la generación y análisis de la seguridad de contraseñas.

Descripción General del Proyecto

La idea central del proyecto es desarrollar una herramienta que evalúe las contraseñas digitadas por el usuario y muestre qué tan seguras son mediante gráficas que ilustren cuánto tiempo tardarían distintos algoritmos de desencriptación en descifrarlas. Además, se busca enseñar a los estudiantes de dónde provienen dichos cálculos y cómo los conceptos vistos en clase, como conteo, permutaciones y combinaciones, se aplican en este contexto.

Componentes y Funcionalidades Clave

    Análisis de Combinaciones:

    • Calcular el número total de combinaciones posibles para la contraseña digitada por el usuario, teniendo en cuenta los tipos de caracteres utilizados (minúsculas, mayúsculas, caracteres especiales y números).
    • Estimar el tiempo que tomaría descifrar la contraseña, asumiendo un algoritmo de ataque por fuerza bruta.
    • Comparar el tiempo que le tomaría a un humano y a una máquina descifrar dicha contraseña.

    Análisis de Ataque (Computadora vs Humano):

    • Suponiendo que un humano puede intentar una contraseña cada 0.5 segundos y que un algoritmo automatizado de fuerza bruta puede hacer entre 1 y 10 microsegundos por intento (1 µs = 0.000001 s), se puede estimar el tiempo necesario para descifrar una contraseña. Utilizando herramientas como JavaScript y la librería Chart.js, es posible visualizar interactivamente estos tiempos según la longitud de la contraseña y la cantidad de combinaciones posibles, lo cual depende de los caracteres utilizados.

Conceptos de Matemáticas Discretas a Aplicar

Este proyecto se fundamenta en los siguientes conceptos vistos en el curso:

  • Principios Fundamentales de Conteo: Principalmente, el principio de la multiplicación, que establece que si hay "n" formas de realizar una primera tarea y "m" formas de realizar una segunda tarea, entonces hay "nm" formas de realizar ambas tareas. Para las contraseñas, cada caracter es uan tarea y el número de opciones es el tamaño del conjunto de caracteres disponibles.
  • Permutaciones y Combinaciones: El orden de los caracteres en una contraseña es importante y los caracteres pueden repetirse, la fórmula clave para calcular el número de combinaciones posibles es N^L, donde N es el número total de caracteres disponibles y L es la longitud de la contraseña.

Ejemplo de Cálculo:

Si el usuario elige una longitud de 10 caracteres y desea incluir minúsculas (26), mayúsculas (26), números (10) y símbolos (aprox. 32), el número total de caracteres disponibles (N) sería 26+26+10+32 = 94. La longitud L es 10.

El número de combinaciones posibles sería 94^10.

        import itertools
        import string
        import time

        def detectar_tipo(password):
            tiene_minus = any(c.islower() for c in password)
            tiene_mayus = any(c.isupper() for c in password)
            tiene_num = any(c.isdigit() for c in password)
            tiene_simbolos = any(c in string.punctuation for c in password)

            alfabeto = ''
            if tiene_num:
                alfabeto += string.digits 
            if tiene_minus:
                alfabeto += string.ascii_lowercase  
            if tiene_mayus:
                alfabeto += string.ascii_uppercase  
            if tiene_simbolos:
                alfabeto += string.punctuation  
            return alfabeto

        def fuerza_bruta_adaptativa(password_objetivo):
            alfabeto = detectar_tipo(password_objetivo)
            print(f"Alfabeto usado: {alfabeto}")
            
            start = time.time()

            for longitud in range(1, len(password_objetivo) + 1):
                for intento in itertools.product(alfabeto, repeat=longitud):
                    intento_actual = ''.join(intento)
